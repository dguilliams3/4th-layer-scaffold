# 4th Layer Engineering Agent Harness

A production workflow harness for AI coding agents. Gives your agent session continuity, structured run tracking, multi-model review, and memory across context compactions and instance swaps.

Works with Claude Code, Cursor, Codex, Gemini, or any agent that reads markdown instructions.

> **For humans:** You don't run any of this yourself. Point your AI coding agent at this scaffold and say "install this into my repo." The agent reads SKILL.md, copies files, configures hooks, and sets up the workflow. Once installed, everything operates automatically as your agent works. You steer; the agents row.

---

## Why This Exists

AI coding agents are stateless by default. Every new session starts cold. Context compacts mid-task. Agents forget what they tried, what failed, and what they decided.

This harness solves that with **externalized agent memory**:

- **SPEC files** capture what success looks like, what's out of scope, and what's been tried
- **TASK_LOG files** capture what actually happened, chronologically
- **HANDOFF docs** capture lessons learned with expiration dates, preventing stale rules from blocking future work
- **Memory promotion** distills durable knowledge from run artifacts into persistent memory files

### "But it creates so many files"

The `runs/` directory is gitignored. Run artifacts are **agent working memory** -- analogous to `__pycache__/`, `node_modules/`, or TypeScript's `dist/` output. They're generated by the agent, for the agent, and never committed. The human doesn't create or maintain them.

Without externalized state, agents retry failed approaches, lose track of multi-step work, and can't hand off to a fresh session. The files aren't overhead -- they're what makes the agent functional across sessions.

## What Gets Installed

| Category             | What                             | Purpose                                                                                                     |
| -------------------- | -------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| **Hooks** (7)        | `.claude/hooks/`                 | Agent type enforcement, git safety, session continuity, subagent protocol, task log reminders               |
| **Skills** (7)       | `skills/` or `~/.claude/skills/` | Task tracking, archival, subagent invocation (Codex/Cursor/Gemini), adversarial review, multi-model council |
| **Docs** (5)         | `docs/coding_agents/`            | Subagent guide, Gemini CLI reference, council protocol, adversarial review, context optimization            |
| **Runtime state**    | `CLAUDE.local.md`                | Active Tasks table + optional Agent Voice (gitignored)                                                      |
| **Workflow pointer** | CLAUDE.md section                | ~116-line slim pointer to the task-tracking skill                                                           |
| **Run infra**        | `runs/CLAUDE-RUNS/`              | Gitignored directory for run artifacts (SPEC, TASK_LOG, HANDOFF)                                            |

### Key Skills

**`task-tracking`** -- Structured run tracking with SPEC versioning, continuous TASK_LOG, HANDOFF docs at completion, and memory promotion on archival. Opt-out, not opt-in.

**`archive-run`** -- Guided archival with metric collection (git/python/node/go/rust), memory candidate triage, and ARCHIVE.md entry generation.

## For Coding Agents: Updating Previously Scaffolded Repos

> **If you are Claude, Gemini, Codex, or another coding agent** and you have already installed this scaffold into a repository, do NOT re-run the full install. Instead:
>
> 1. **Check what changed** -- diff this repo against the version you last installed
> 2. **Apply only the diffs** -- update changed files, add new ones
> 3. **Don't overwrite customized files** -- if a file has filled-in `[TODO]` markers or project-specific content, merge carefully

## Design Principles

- **Task tracking is a skill, not inline.** The full protocol (~540 lines) lives in `task-tracking/SKILL.md`, loaded on-demand. CLAUDE.md gets a ~116-line pointer. Saves ~2,500 tokens per session that doesn't need tracking.
- **Active Tasks live in CLAUDE.local.md.** Gitignored in target repos -- no merge conflicts, no git noise. Same system prompt visibility as CLAUDE.md.
- **Opt-out, not opt-in.** The `session-run-prompt` hook enforces task-tracking by default. Disable per-repo with `.claude/task-tracking.disabled`.
- **Run artifacts are ephemeral.** The `runs/` directory is gitignored. Like build output, it's generated by the toolchain and never committed.
